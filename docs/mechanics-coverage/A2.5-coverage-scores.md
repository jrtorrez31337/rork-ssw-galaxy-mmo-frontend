## 8. Technical Debt & Recommendations

### Code Quality Issues

1. **Duplicate SSE Connections** - Each hook opens separate EventSource
   - **Recommendation**: Create singleton SSE manager
2. **Hardcoded Fanout URL** - Direct IP in multiple hooks
   - **Recommendation**: Use config.FANOUT_URL consistently
3. **Missing API Client** - Chat has no API client
   - **Recommendation**: Add `api/chat.ts`
4. **Inconsistent Error Handling** - Some APIs use try/catch, others don't
   - **Recommendation**: Standardize error handling wrapper

### Performance Issues

1. **Multiple SSE Connections** - One per event type
   - **Impact**: Network overhead, battery drain
   - **Fix**: Multiplex all events through single connection
2. **No Query Caching** - React Query invalidation is aggressive
   - **Impact**: Excessive API calls
   - **Fix**: Add staleTime, cacheTime to queries
3. **No Optimistic Updates** - All mutations wait for server
   - **Impact**: UI feels slow
   - **Fix**: Add optimistic updates for common actions

### Security Issues

1. **No Token Refresh** - Access tokens expire after 15 minutes
   - **Impact**: Users get logged out mid-session
   - **Fix**: Implement auto-refresh with refresh token
2. **Hardcoded IPs** - Direct Fanout IP in hooks
   - **Impact**: Won't work in production
   - **Fix**: Use Gateway proxy for all SSE
